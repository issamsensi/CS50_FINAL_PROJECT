SECTION 4: TECHNICAL EXPLANATION (60-90 seconds)
================================================================================

[SWITCH TO: Code editor OR back to camera]

"Now let me explain the technical implementation.

For the GAME component, I wrote over 600 lines of Python code organized into 
six classes:
- The Game class manages game states and the main loop
- The Player class handles movement, shooting, and upgrades
- The Enemy class with three variants using inheritance
- The Bullet class with trigonometry for directional movement
- The Collectible class for coins and crystals
- And the Particle class for visual effects

I used Pygame for rendering, collision detection, and event handling. The 
bullet trajectories use sine and cosine functions from Python's math module 
to calculate velocity based on angle.

For the WEBSITE component, I created a Flask application with five routes:
- The index route for the home page
- About, Developer, and Contact pages
- And a newsletter route that handles POST requests

The frontend uses Jinja2 template inheritance - I have a base layout.html with 
the header and footer, and all other pages extend from it. This keeps my code 
DRY - Don't Repeat Yourself.

I styled everything with custom CSS using CSS variables for consistent theming, 
and integrated Bootstrap 5 for responsive components like the carousel and forms.

The newsletter system uses SQLite with the CS50 SQL library. When someone 
subscribes, I check if their email already exists to prevent duplicates, then 
insert it into the database.

I also used PyInstaller to compile the game into a standalone executable that 
can run on Windows and Linux without requiring Python to be installed."

SECTION 5: CHALLENGES & LEARNING (45-60 seconds)
================================================================================

[BACK TO: Camera OR slides]

"Throughout this project, I faced several interesting challenges.

One of the biggest was implementing the collision detection between bullets and 
enemies. I had to carefully manage lists and remove objects during iteration, 
which can cause errors if not done correctly. I solved this by iterating over 
copies of the lists.

Another challenge was the particle system. I wanted smooth, natural-looking 
explosions, so I implemented a system where each particle has random velocity, 
gradually shrinks, and fades over time.

For the website, implementing the newsletter system taught me about database 
operations, form validation, and user feedback with flash messages.

I also learned about responsive web design - making sure the website looks good 
on all screen sizes using media queries and flexbox.

This project really pushed me to combine multiple technologies and concepts from 
CS50: object-oriented programming, web development, databases, algorithms, and 
user interface design."
